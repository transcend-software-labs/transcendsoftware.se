# Intake playbook

The clarifying questions asked before any planning or building. This is
load-bearing: it's what separates this from a tool that confidently builds the
wrong thing. Ask only the few highest-value questions — the ones that would most
change the result if guessed wrong. Max 3.

## The questions that usually matter most

1. **Brochure or shop?** "Do you want customers to buy online, or just see the
   site and contact you?" — decides static vs. database + storage + their Stripe.
2. **Assets.** "Do you have your own photos and logo, or should we use
   placeholders for now?" — the one thing the agent can't truthfully invent.
3. **Language(s).** Often Swedish; sometimes also English.
4. **Key pages / must-haves.** Only if the brief is vague about scope.
5. **Existing brand/material.** Colours, an existing site, a name for the
   business.

## Worked example — the äppelmust site

Brief: *"A website for my neighbour's apple farm to sell äppelmust."*

Ask:
- Buy online, or contact/visit to order?
- Do you have photos of the orchard and bottles, or use placeholders?
- Swedish only, or also English?

That's enough to choose brochure-vs-shop, plan the pages, and know what content
is missing — without a long form.

## Rules

- Plain language, no jargon. The customer is non-technical.
- If the brief is already complete, ask nothing and proceed.
- Don't ask what you can sensibly default (then note the default).
